Projeto de TCC

Aplicação Web para Dados da Plataforma Nilo Peçanha

Uma solução baseada em engenharia de dados, API e interface Web para apoiar a análise de indicadores da Educação Profissional, Científica e Tecnológica.

O projeto propõe automatizar a ingestão, o tratamento, a consolidação e a disponibilização de dados da Plataforma Nilo Peçanha, facilitando consultas, análises, exportações e geração de insights.

Sobre o TCC

A Plataforma Nilo Peçanha reúne dados importantes sobre a Rede Federal de Educação Profissional, Científica e Tecnológica. Porém, para uso analítico, esses dados podem exigir coleta, organização, padronização e estruturação.

Problema

Dados públicos podem estar distribuídos em diferentes formatos e exigir etapas manuais antes de serem usados em análises.

Proposta

Criar uma aplicação Web capaz de coletar, tratar, consolidar e disponibilizar os dados de forma mais organizada.

Resultado Esperado

Apoiar consultas, indicadores, dashboards, exportações e consumo dos dados tratados por meio de uma API.

Funcionalidades Previstas

A solução proposta organiza o ciclo dos dados desde a origem até a disponibilização para consulta, análise e exportação.

Ingestão de Dados

Coleta de dados da Plataforma Nilo Peçanha e, futuramente, de outras fontes oficiais ou arquivos complementares.

Tratamento e Padronização

Limpeza, validação, transformação e organização dos dados para melhorar sua consistência e uso analítico.

API de Consumo

Disponibilização dos dados tratados por meio de endpoints para consulta por sistemas, dashboards ou aplicações.

Interface Web

Visualização de indicadores, filtros, tabelas e informações consolidadas por meio de uma aplicação Web.

Exportação de Dados

Possibilidade de exportar dados tratados em formatos como CSV, JSON ou outros formatos definidos no projeto.

Rastreabilidade

Registro de cargas, origem dos dados, falhas, status de processamento e informações de monitoramento.

Tecnologias Previstas no Projeto de TCC

As tecnologias abaixo se referem ao sistema completo proposto no TCC, e não apenas a esta landing page acadêmica.

  • Python e FastAPI: back-end, API e integração.
  • PostgreSQL: armazenamento dos dados tratados e consolidados.
  • MongoDB ou staging: camada para dados brutos ou semiestruturados.
  • Engenharia de Dados: ingestão, validação, limpeza e transformação.
  • Interface Web: consulta, filtros, indicadores e visualização.
